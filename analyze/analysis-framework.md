# 業務分析フレームワーク 詳細

## 分類の判定フロー

```
業務を1つ取り出す
  ├─ 手順が定型化されている？
  │   ├─ Yes → 定常業務
  │   └─ No → 発生が予測可能？
  │       ├─ Yes（低頻度だが手順あり）→ 定常業務
  │       └─ No → 例外業務
  └─ 本質的に人間の判断・感情対応が必要？
      ├─ Yes → 対人業務
      └─ No → 上記フローで判定
```

対人業務の周辺作業（議事録、日程調整、資料準備など）は**別の業務として切り出し**、定常業務として再分類する。

## 費用対効果スコアの算出

```
年間工数 = 頻度（回/年） × 1回あたりの時間（時間）
自動化可能割合 = 0〜100%（業務内容に基づき推定）
削減見込み = 年間工数 × 自動化可能割合

ランク判定:
  削減見込みの規模感に応じてA/B/Cにランク分けする。閾値は対象の業務規模に合わせて調整する
```

### 自動化可能割合について

自動化可能割合は業務の具体的な内容次第で大きく変動する。固定値ではなく、対象業務の実態に基づいて推定すること。

## 優先順位の決め方

ランクだけでなく、以下も加味して最終的な優先度を決める:

1. **業務削減（廃止・統合・頻度削減）**: コストゼロで効果が出るもの → 最優先
2. **クイックウィン**: 導入難易度「低」× ランクA → 次に優先
3. **ROI重視**: 削減時間が大きい順
4. **波及効果**: 1つのツール導入で複数業務が改善されるもの
5. **ユーザーの困り度**: ヒアリングで「しんどい」と言われた業務を優先
