# 業務抽出パターン

Step 1（業務内容の自動抽出）とStep 2（ギャップ確認ヒアリング）の詳細ガイド。

---

## ドキュメント形式別の抽出アプローチ

### Excel / CSV

最もよくある入力形式。構造化されているが、フォーマットは千差万別。

**抽出手順**:
1. ファイルを読み込み、シート一覧・列名を確認
2. 業務名に相当する列を特定（「業務名」「作業内容」「タスク」「業務項目」など）
3. 頻度・時間・担当者に相当する列を特定
4. 空行・結合セル・ヘッダー行を考慮して抽出
5. 抽出結果を一覧表で提示

**よくある罠**:
- セル結合で親子関係が表現されている（「営業部」の下に複数業務）→ 結合を解除して個別業務に展開
- 「備考」欄に重要な情報がある（「月末は3倍の量」等）→ 備考も取得する
- 1行に複数業務が書かれている（「メール確認・返信」）→ 分割するかユーザーに確認

### PDF

手順書、業務フロー図、組織図など。

**抽出手順**:
1. テキスト抽出を試みる
2. テキストが取れない場合はOCR（画像として読み取り）
3. 業務フロー図の場合は、フロー内の各ボックスを業務として抽出
4. 手順書の場合は、見出しレベルで業務を切り出す

### Word / テキスト

議事録、業務記述書、引き継ぎ資料など。

**抽出手順**:
1. 見出し構造を解析（Wordのスタイル or マークダウン的な構造）
2. 業務に関する記述を抽出（動詞+目的語のパターン: 「〜を作成する」「〜を確認する」）
3. 頻度・時間の記述を紐付け
4. 非構造的な文章からは、箇条書きや表の部分を優先的に抽出

### 画像（ホワイトボード写真、手書きメモ等）

**抽出手順**:
1. 画像を読み取り、テキスト・図形を認識
2. 付箋やフロー図のボックスを業務として抽出
3. 手書きで読みにくい部分は「〇〇（判読困難）」として残す
4. ユーザーに確認を取る

### ドキュメントなし（ヒアリングのみ）

Step 2のヒアリングから直接始める。この場合は以下の構造化質問で業務を引き出す。

---

## ヒアリングテンプレート

### 初回質問（業務の洗い出し）

ドキュメントがない場合、または抽出結果の補完に使う。

```
まず全体像を把握させてください。

1. 普段のお仕事で「毎日やること」を教えていただけますか？
   （朝出勤してから退勤するまでの流れでざっくりで大丈夫です）

2. 「毎週」「毎月」など定期的にやっている業務はありますか？

3. 使っているツールやシステムはどんなものがありますか？
   （Excel、メール、社内システムなど）
```

ユーザーの回答に対して「それは大体どのくらい時間かかりますか？」「それは週に何回くらいですか？」と自然に深掘りする。一度に全部聞かず、会話のリズムを大事にする。

### ギャップ確認質問

Step 1で抽出した一覧を提示した後に使う。

```
一覧を見ていただいてありがとうございます。いくつか確認させてください。

- この中で「実際はもっと時間がかかっている」ものはありますか？
- 逆に「これはそんなに大変じゃないよ」というものは？
- 一覧に載っていないけど、地味に時間を取られている作業ってありますか？
- イレギュラーで発生して困るような業務はありますか？
```

### しんどさの深掘り

```
ありがとうございます。もう少しだけ聞かせてください。

- 一番「なんとかしたい」と感じている業務はどれですか？
- その業務の何が大変ですか？（量が多い？複雑？面倒？）
- もし魔法で1つだけ自動化できるとしたら、どれを選びますか？
```

この質問は提案の優先順位付けに直結する。ユーザーの主観的な「しんどさ」は、数値化できない重要な情報。

## 抽出結果の整理フォーマット

ドキュメント抽出・ヒアリングの結果を以下の形式でユーザーに提示する。

```markdown
## 業務一覧（暫定）

| # | 業務名 | 頻度 | 所要時間/回 | 関係者 | 使用ツール | メモ |
|---|--------|------|------------|--------|-----------|------|
| 1 | メール確認・返信 | 日次 | 30分 | 本人 | Outlook | 定型返信が多い |
| 2 | 週次報告書作成 | 週次 | 2時間 | 本人→上長 | Excel, PowerPoint | データ集計に時間 |
| 3 | ... | ... | ... | ... | ... | ... |

> この一覧で抜け漏れや、実態と違う部分はありますか？
```

**ポイント**:
- 「メモ」列に、自動化のヒントになりそうな情報を入れておく
- 所要時間が不明な場合は「?」にして、ユーザーに聞く
- 多すぎる場合（20業務以上）は、カテゴリで区切って見やすくする

## よくある業務パターン（抜け漏れ防止用チェックリスト）

ヒアリング時に「この中でやっているものはありますか？」と確認に使う。全部聞くのではなく、文脈に合わせて抜け漏れそうなものだけピックアップする。

### バックオフィス系
- [ ] 経費精算・領収書処理
- [ ] 請求書の作成・送付
- [ ] 勤怠管理・入力
- [ ] 備品・消耗品の発注
- [ ] 契約書の管理・更新確認

### 情報管理系
- [ ] ファイルの整理・命名・保存
- [ ] マニュアル・手順書の更新
- [ ] 顧客情報・案件情報のCRM入力
- [ ] ナレッジベースの更新

### コミュニケーション系
- [ ] 会議の日程調整
- [ ] 議事録の作成・共有
- [ ] 社内チャットでの問い合わせ対応
- [ ] 定例報告（日報、週報、月報）

### 分析・モニタリング系
- [ ] KPIデータの集計・ダッシュボード更新
- [ ] アクセス解析・レポート
- [ ] 在庫・売上データの確認
- [ ] 競合・市場情報の収集
