# ヒアリング・情報収集フロー詳細

Step 1（自動情報収集）とStep 2-3（ヒアリング）の詳細手順。

## AskUserQuestion の使い方

- ヒアリングには必ず `AskUserQuestion` ツールを使用する
- 1回の呼び出しで最大4問まで同時に聞ける
- 事務的な質問（許可取得、基本情報）はまとめて効率化する
- **ただし、業務の分解に必要な深掘りは回数を制限しない** — 「どの業務のどの工程を自動化するか」の特定がこのスキルの核心。業務を工程レベルまで分解してボトルネックを見つけるために、追加の質問を柔軟に行う
- 選択肢のlabelは簡潔に、descriptionで補足


---

## 第1回 AskUserQuestion（Step 1b: 許可 + 基本情報）

ツール検出直後に、**許可取得と基本ヒアリングを1回にまとめる**。

### 検出ツールがある場合（4問同時）

```
質問1 [接続済みツール]:
  header: "情報収集の許可"
  question: "以下のツールが接続可能です。業務を把握するために読み取りを許可するものを選んでください（選択したもの＝許可。何も選ばなければ許可なし）"
  multiSelect: true
  ⚠️ 選択肢には検出されたツール名のみを並べる。「許可しない」という選択肢は設けない。
     ユーザーが何も選ばずにOtherで「なし」等と答えた場合＝全ツール不許可として扱う。
  options: 検出されたツール＋ブラウザ履歴をそれぞれ1つずつ:
    label: "Slack" 等（ツール名のみ。「許可する」は付けない）
    description: ツールごとに読み取る情報とセキュリティ注意点を記載

    ブラウザ履歴が検出された場合の選択肢例:
    label: "ブラウザ履歴"
    description: "読み取る情報: 直近2週間の閲覧サイトURL・アクセス頻度。
                  ⚠️ 閲覧履歴にはプライベートなサイトが含まれる場合があります。
                  業務で使っているツール・サービスの特定に使用します。データはこのセッション内のみで使用します。"

質問2 [追加ツール]:
  header: "追加接続"
  question: "他にも接続すると業務の把握がしやすくなります。新たに接続したいものはありますか？"
  multiSelect: true
  options: 未接続のツール候補:
    - Google Drive: "ドキュメントやスプレッドシートから業務内容を把握できます"
    - Notion: "ページやデータベースから業務構造を把握できます"
    - Discord: "チャンネルの会話から業務パターンを把握できます"
    - カレンダー: "会議パターンや定例の頻度を把握できます"
    - 特になし: "追加接続は不要です"

質問3 [対象規模]:
  header: "対象規模"
  question: "この改善の対象はどの規模ですか？"
  multiSelect: false
  options:
    - 個人の業務: "自分1人の業務を効率化したい"
    - チーム全体: "チームメンバー全員の業務を改善したい"
    - 会社全体: "部門横断・全社的にAI導入を進めたい"

質問4 [制約条件]:
  header: "制約条件"
  question: "気になる制約はありますか？"
  multiSelect: true
  options:
    - セキュリティ重視: "データを外部に出すことに抵抗がある・社内規定が厳しい"
    - 予算が限られている: "新しい有料ツールの導入は難しい"
    - ITスキルが低め: "自分やメンバーのITリテラシーに不安がある"
    - 特になし: "制約は特に気にしていない"
```

### 検出ツールが0件の場合（3問同時）

質問1を省略し、質問2-4を表示。「現在接続されているツールがありません」と伝えた上で提示。

### 追加ツールを選んだ場合

接続方法を簡潔にガイドし、設定完了後に情報収集を実行する。
ユーザーが「後で設定する」「やっぱり不要」と言った場合は、そのまま次に進む。
**追加ツール設定のために別のAskUserQuestionを挟まない**（テキストでガイドし、ユーザーの返答を待つ）。

## 自動収集 → 業務一覧の提示（Step 1c → Step 2）

許可されたツールで情報収集を実行し、結果を業務一覧として整理してテキストで提示する。

```
収集結果から以下を整理:
■ 推定される業務一覧（テーブル形式で提示）
  - 業務名、推定頻度、情報ソース
■ 推定されるツール環境
■ 判明しなかった情報
```

自動収集がない場合（ツール未接続）はこのステップをスキップし、直接第2回ヒアリングに進む。

---

## 自動収集から仮説を立てる（Step 2 → Step 3 の橋渡し）

業務一覧を提示する前に、収集データから**仮説**を立てる。ヒアリングはこの仮説の検証として行う。

### 仮説の立て方

収集した生データの中に、以下のシグナルを探す:

- **高頻度アクセス × ツール間の移動**: ブラウザで同じツールを何度も行き来している → そのツール間の連携が手動のまま
- **Slackの報告パターン**: 毎日同じ形式で「○○：△本」と報告 → 定量管理が手動で回っている
- **議事録に繰り返し出る話題**: 同じ課題が何度も議題に上がる → 根本解決されていない構造的問題
- **ブラウザで高頻度だがヒアリングで触れられない**: 当たり前すぎて意識されていない → 無意識の時間消費

### 仮説の提示方法

業務一覧の後に、テキストで仮説を提示してからヒアリングに入る:

```
「収集した情報を見ると、○○に相当な時間が使われているようです。
 特に△△の部分が手動で残っているのではないかと推測しているのですが、
 もう少し詳しく聞いてもいいですかね？」
```

仮説が外れたら柔軟に修正し、新たな問いを立てる。仮説を押し付けない。

## 第2回 AskUserQuestion（Step 3: 仮説検証ヒアリング）

自動収集の結果によって質問内容が変わる:

### 自動収集ありの場合（確認型）

```
質問1 [業務確認]:
  header: "業務の確認"
  question: "〇〇から以下の業務を推定しました。この中で実際にやっている業務を選んでください"
  multiSelect: true
  options: 推定した業務を選択肢に（最大4つ。5つ以上ある場合は主要なものに絞る）

質問2 [しんどい業務]:
  header: "困りごと"
  question: "この中で特に時間がかかる・面倒だと感じる業務はどれですかね？"
  multiSelect: true
  options: 推定した業務から選択肢を生成（最大4つ）

質問3 [所要時間]:
  header: "作業時間"
  question: "1日の業務時間のうち、一番時間を使っている作業はざっくりどのくらいですか？"
  multiSelect: false
  options:
    - 1日の半分以上: "4時間以上かかっている作業がある"
    - 1日の1/4くらい: "2時間程度かかる作業がいくつかある"
    - 合間合間に: "30分〜1時間の作業が断続的に入る"
    - ばらばら: "日によって全然違う"
```

### 自動収集なしの場合（聞き取り型）

```
質問1 [業務内容]:
  header: "業務内容"
  question: "主な業務はどんな内容ですか？"
  multiSelect: true
  options:
    - 記事・コンテンツ制作: "記事の執筆・編集・ファクトチェック・リライトなど"
    - データ集計・レポート: "Excel集計、ダッシュボード、報告書作成など"
    - コミュニケーション: "メール・チャット対応、会議、日報など"
    - その他: "上記以外（後で詳しくお聞きします）"

質問2 [困りごと]:
  header: "困りごと"
  question: "特に「これがしんどい、何とかしたい」と感じている作業はありますか？"
  multiSelect: true
  options:
    - 定型作業の繰り返し: "毎回同じような作業を手動でやっている"
    - 情報の整理に時間がかかる: "散らばった情報を1箇所にまとめる作業が多い"
    - 会議後の作業が重い: "議事録作成やタスク整理に時間がかかる"
    - 確認・チェックが多い: "ファクトチェックや品質確認に時間を取られる"

質問3 [作業時間]:
  （自動収集ありの場合と同じ）

質問4 [ツール環境]:
  header: "使用ツール"
  question: "普段使っているツール・環境はどれですか？"
  multiSelect: true
  options:
    - Microsoft 365: "Outlook, Teams, SharePoint, Power Automate など"
    - Google Workspace: "Gmail, Google Drive, Sheets, Apps Script など"
    - Slack: "チームコミュニケーションにSlackを使用"
    - その他: "Notion, Discord, 社内システムなど"
```

---

## ヒアリングのトーン

- **自動収集で既に分かっていることは聞かない**
- 「Slackを見ると〇〇をされているようですが…」のように、すでに理解していることを示す
- 質問文は「〜ですかね？」くらいの柔らかさで
- 選択肢の説明は具体例を入れて分かりやすく

## 抜け漏れ防止チェックリスト

ヒアリング結果を確認する際に、以下の業務カテゴリが抜けていないかチェック:

- [ ] バックオフィス（経費精算、請求書、勤怠）
- [ ] 情報管理（ファイル整理、マニュアル更新、CRM入力）
- [ ] コミュニケーション（日程調整、議事録、日報・週報）
- [ ] 分析・モニタリング（KPI集計、アクセス解析、在庫確認）
