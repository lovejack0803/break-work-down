# Excel (.xlsx) 生成ガイド

## 概要

HTMLレポートの数値データを `.xlsx` ファイルとしてエクスポートする。ユーザーが数字を自分で調整・試算できる「触る資料」。ExcelおよびGoogle スプレッドシートの両方で開ける。

**HTMLレポートとの情報量の一致が必須。** HTMLにあるセクションは全てXLSXにも含める。XLSXだけ情報が少ない状態にしてはならない。

## いつ生成するか

以下の場合に**提案**する:
- 「数字をいじりたい」「試算を自分で調整したい」と言われた場合
- 業務一覧やROI試算をデータとして手元に残したい場合
- 「上に共有したい」「稟議に使いたい」と言われた場合

## 生成方法

`openpyxl` を使用してPythonで生成する。

```bash
pip install openpyxl
```

## シート構成（7シート）

HTMLレポートの構成と1対1で対応させる。

| シート | HTMLセクション | 必須/任意 |
|---|---|---|
| 1. 業務一覧 | 1. 業務の全体像（トップ5テーブル） | 必須 |
| 2. 業務削減提案 | 2. 業務削減提案（廃止/統合/頻度削減） | 必須 |
| 3. 仕組み化候補トップ3 | 3. 仕組み化候補トップ3 | 必須 |
| 4. 1位の深掘り | 4. 1位の深掘り（現在vs改善 + 始め方） | 必須 |
| 5. 使用ツール | 5. 使用ツール + 7. 簡易代替案 | 必須 |
| 6. つまずきポイント | 6. つまずきやすいポイント | 必須 |
| 7. 削減効果の試算 | 8. まとめ（KPI・試算） | 必須 |

---

### シート1: 業務一覧

業務トップ5とその分析結果。

| 列 | 内容 | 備考 |
|----|------|------|
| A | # | 連番 |
| B | 業務名 | |
| C | 頻度 | 日次/週次/月次 |
| D | 1回あたりの時間 | 分単位 |
| E | 月間回数 | |
| F | 月間合計時間 | **計算式**: `=D*E/60` |
| G | 分類 | 定常/例外/対人 |
| H | 自動化余地 | A/B/C |
| I | メモ | ボトルネックの説明等 |

最終行に合計行（`=SUM(F2:F末尾)`）を入れる。

### シート2: 業務削減提案

Step 4 で検討した廃止・統合・頻度削減の一覧。「根拠」列を必ず含める。

| 列 | 内容 |
|----|------|
| A | 対象業務 |
| B | 削減アプローチ（廃止/統合/頻度削減） |
| C | 具体的な内容 |
| D | 根拠（データや観察事実に基づく理由） |
| E | 想定削減効果 |
| F | 進め方（段階的な実施計画） |

### シート3: 仕組み化候補トップ3

業務削減で対処できない残存業務の仕組み化提案。順位ごとに行背景色を変える（1位=薄いピーチ、2位=薄いセージ、3位=薄いプラム）。

| 列 | 内容 |
|----|------|
| A | 順位 |
| B | 対象業務 |
| C | 現状の問題 |
| D | 改善後のイメージ |
| E | 現状時間/月 |
| F | 改善後時間/月 |
| G | 削減効果/月（**計算式**: `=E-F`） |
| H | 推奨ツール |
| I | 導入難易度（低/中/高） |

### シート4: 1位の深掘り

2つのセクションで構成する。

**セクション1: 現在 vs 改善後の比較表**

| 列 | 内容 |
|----|------|
| A | 比較項目（ステップ数、1件あたりの時間、タイムラグ、リスク等） |
| B | 現在のフロー |
| C | 改善後のフロー |

改善後の列はセージ色のフォントで視覚的に区別する。

**セクション2: 始め方の3ステップ**（比較表の下に配置）

| 列 | 内容 |
|----|------|
| A | ステップ番号 |
| B | やること |
| C | 所要時間 |
| D | 詳細 |

### シート5: 使用ツール

HTMLの「使用ツール」と「簡易代替案」を1つのテーブルに統合する。

| 列 | 内容 |
|----|------|
| A | ツール名 |
| B | 用途 |
| C | コスト |
| D | 選定理由 |
| E | 代替手段 |
| F | 代替メモ |

### シート6: つまずきポイント

| 列 | 内容 |
|----|------|
| A | よくある困りごと（太字） |
| B | 背景 |
| C | 回避策 |

### シート7: 削減効果の試算

ユーザーが値を変えると結果が自動更新される設計。**業務削減と仕組み化を分離して表示する。**

```
■ 構成

  タイトル: 「業務改善 削減効果の試算」
  説明: 「黄色のセルの値を変更すると、削減効果が自動で再計算されます。」

  ── 業務削減（ツール不要）──
  [ヘッダー行]
  項目1...
  項目N...
  小計（業務削減）

  ── 仕組み化（ツール導入）──
  [ヘッダー行]
  項目1...
  項目N...
  小計（仕組み化）

  総合計
```

各行の列構成:

| 列 | 内容 | 備考 |
|----|------|------|
| A | 対象業務 | |
| B | 1回の時間(分) | **入力セル**（黄色背景） |
| C | 月間回数 | **入力セル** |
| D | 自動化率(%) | **入力セル**。コメントで説明を付与 |
| E | 定着率(%) | **入力セル**。デフォルト70% |
| F | 月間削減(時間) | **計算式**: `=B*C*(D/100)*(E/100)/60` |
| G | 年間削減(時間) | **計算式**: `=F*12` |

小計は `=SUM(F開始:F末尾)`。総合計は `=小計1+小計2`。

## スタイル

```python
from openpyxl.styles import Font, PatternFill, Alignment, Border, Side

# カラーパレット（HTMLレポートと統一）
HEADER_BG = "2C2C34"      # charcoal
HEADER_FG = "FFFFFF"
ACCENT = "E07A5F"         # peach
SAGE = "6B8F71"           # sage
PLUM = "8A6C8A"           # plum
INPUT_BG = "FFF8E1"       # 薄い黄色（入力セル）
BORDER_COLOR = "E8E2DA"
LIGHT_PEACH = "FDF0EB"    # 1位の行背景
LIGHT_SAGE = "EEF4EF"     # 2位の行背景
LIGHT_PLUM = "F3EEF3"     # 3位の行背景

# ヘッダー行
header_font = Font(name="Yu Gothic", bold=True, color=HEADER_FG, size=10)
header_fill = PatternFill(start_color=HEADER_BG, end_color=HEADER_BG, fill_type="solid")

# 本文
body_font = Font(name="Yu Gothic", size=10)

# 入力セル
input_fill = PatternFill(start_color=INPUT_BG, end_color=INPUT_BG, fill_type="solid")

# 罫線
thin_border = Border(
    bottom=Side(style="thin", color=BORDER_COLOR)
)

# 順位別のアクセントフォント
accent_font = Font(name="Yu Gothic", size=10, bold=True, color=ACCENT)  # 1位 / 業務削減小計
sage_font = Font(name="Yu Gothic", size=10, bold=True, color=SAGE)      # 2位 / 仕組み化小計
plum_font = Font(name="Yu Gothic", size=10, bold=True, color=PLUM)      # 3位
```

## 重要な注意点

- **HTMLレポートと情報量を一致させる** — HTMLにあるセクションは全てXLSXにも含める。数値の齟齬も防ぐ
- **Pythonで計算してハードコードしない** — 削減時間や年間効果などの数値は、必ずExcelの数式（`=D2*E2/60` 等）で記述する。Pythonで計算した結果をセルにベタ書きすると、ユーザーが入力値を変えても結果が連動しない
- **数式エラーをゼロにする** — 生成後に `#REF!`、`#DIV/0!`、`#VALUE!`、`#N/A`、`#NAME?` が含まれていないことを確認する。特に行の増減で参照がずれるケースに注意
- **入力セルを視覚的に区別する** — 薄い黄色背景 + セルコメントで「この値を変更できます」と案内
- **ファイル名は日本語で** — `業務分析レポート.xlsx` のように、何のファイルか一目で分かる名前にする
- **順位の色分けを統一する** — 1位=peach、2位=sage、3位=plumはHTML・XLSXで共通
