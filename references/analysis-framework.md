# 業務分析フレームワーク 詳細

## 分類の判定フロー

```
業務を1つ取り出す
  ├─ 手順が定型化されている？
  │   ├─ Yes → 定常業務
  │   └─ No → 発生が予測可能？
  │       ├─ Yes（低頻度だが手順あり）→ 定常業務
  │       └─ No → 例外業務
  └─ 本質的に人間の判断・感情対応が必要？
      ├─ Yes → 対人業務
      └─ No → 上記フローで判定
```

対人業務の周辺作業（議事録、日程調整、資料準備など）は**別の業務として切り出し**、定常業務として再分類する。

## 費用対効果スコアの算出

```
年間工数 = 頻度（回/年） × 1回あたりの時間（時間）
自動化可能割合 = 0〜100%（業務内容に基づき推定）
削減見込み = 年間工数 × 自動化可能割合

ランク判定:
  削減見込み ≥ 50時間/年 → A
  10時間/年 ≤ 削減見込み < 50時間/年 → B
  削減見込み < 10時間/年 → C
```

### 自動化可能割合の目安

| 業務タイプ | 自動化可能割合の目安 |
|-----------|-------------------|
| データ入力（定型フォーマット） | 80〜95% |
| レポート作成（定型） | 70〜90% |
| メール対応（定型返信） | 50〜70% |
| 日程調整 | 60〜80% |
| 議事録作成 | 70〜85% |
| データ分析・集計 | 60〜80% |
| 承認フロー管理 | 40〜60% |
| 問い合わせ対応（社内FAQ） | 50〜70% |
| 顧客折衝・交渉 | 5〜15%（周辺作業のみ） |
| 企画・戦略立案 | 10〜20%（情報収集部分のみ） |

## 複合業務の分解

1つの業務名の中に複数の作業が含まれている場合は分解する。

**例**: 「月次報告書の作成」
- データ収集（定常・A） → スプレッドシート自動集計
- グラフ作成（定常・A） → BI ツール自動生成
- 考察執筆（対人・B） → AI下書き + 人間レビュー
- 上長への報告（対人・C） → そのまま

## 業務削減の判定（Step 4で使用）

自動化可能割合を算出する**前に**、以下の削減判定を行う:

| 判定 | 基準 | 例 |
|------|------|-----|
| 🗑️ 廃止 | 成果物が誰にも使われていない／形骸化している | 誰も読まない日次報告、重複する承認ステップ |
| 🔗 統合 | 同じデータ・目的の業務が複数存在 | CRMとスプレッドシートの二重管理 |
| 📉 頻度削減 | 現在の頻度が成果に対して過剰 | 毎日→週次、全件→抽出チェック |
| 🔧 仕組み化 | 上記で対処不可、ITツールが必要 | 定型データ入力の自動化 |

**廃止・統合・頻度削減はツール導入コストゼロ。** ROI計算で「導入コスト0、削減効果あり」として最上位に位置づける。

## 優先順位の決め方

ランクだけでなく、以下も加味して最終的な優先度を決める:

1. **業務削減（廃止・統合・頻度削減）**: コストゼロで効果が出るもの → 最優先
2. **クイックウィン**: 導入難易度「低」× ランクA → 次に優先
3. **ROI重視**: 削減時間が大きい順
4. **波及効果**: 1つのツール導入で複数業務が改善されるもの
5. **ユーザーの困り度**: ヒアリングで「しんどい」と言われた業務を優先
