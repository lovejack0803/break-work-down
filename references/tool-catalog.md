# AI・自動化ツールカタログ

Step 6（シンプルな仕組みの提案）で使用する。業務カテゴリ別に適用可能なツールを整理し、提案の質と一貫性を担保する。

**このカタログは執筆時点のスナップショットであり、すぐに陳腐化する。** AI・自動化ツールは数週間〜数ヶ月単位で機能・料金・選択肢が変わる。カタログの記載をそのまま提案に使わず、**提案のたびにWebSearchで最新情報を確認すること**が前提。ここに書いてあるツールが最適解とは限らないし、ここに書いていないツールが最適解である可能性も常にある。

## ツール選定の原則

1. **段階的に提案する**: いきなり高機能ツールを提案しない。「まず無料で試せるもの → 本格導入するならこれ → さらに高度な自動化を目指すならこれ」の3段構成で提案する。
2. **既存環境と新しい選択肢の両方を検討する**: 既存ツールの拡張も、新しいツールの導入も、フラットに比較する。「○○を使っているから△△」という短絡的な提案をしない。
3. **無料→低コスト→有料の順に検討**: 同じことができるなら安い方を提案する。有料ツールを提案する場合は、無料代替の存在も併記する。
4. **組み合わせを意識する**: 1つのツールで複数の業務を改善できるなら、個別ツールを複数入れるより優先する。
5. **ITリテラシーに合わせる**: ヒアリングで得たユーザーのスキルレベルに応じて、ノーコード/ローコード/コードベースの提案を使い分ける。

## カテゴリ別おすすめフロー

各業務カテゴリで「まずこれ → 次にこれ」の順で提案する。**ただし、WebSearchで確認した最新の選択肢があればそちらを優先する。**

### 1. データ入力・転記

## 時間削減の見積もり方法

削減時間は**控えめに見積もる**。過大な期待を持たせると信頼を失う。

### 見積もりの公式

```
月間削減時間 = 現状の月間作業時間 × 自動化率 × 定着率係数
```

- **自動化率**: references/analysis-framework.md の目安を使う
- **定着率係数**: 導入直後は 0.5〜0.7（慣れるまで時間がかかる）、定着後は 0.8〜0.9

### レンジで提示する

「月8時間削減」ではなく「月6〜10時間の削減見込み」のようにレンジで示す。根拠も1行で添える。

```
例: 「月次レポートのデータ集計部分を自動化することで、月6〜10時間の削減が見込まれます
    （現状: 月12時間 × 自動化率70% × 定着率0.7〜0.9）」
```

### やってはいけない見積もり

- 「AI導入で業務時間が半減」のような大風呂敷
- 対人業務の本質部分（交渉、判断）の時間を削減対象に含める
- 学習コスト・移行コストを無視した見積もり

## 導入手順テンプレート

各提案の「実装手順」は以下の構成で3〜5ステップにまとめる。

```
1. 準備: アカウント作成 / ツール導入 / 権限設定
2. 設定: ワークフロー構築 / テンプレート作成 / API連携
3. テスト: 小規模で試行（1週間程度）
4. 定着: 2週間後に振り返り / 微調整
```

## 提案の組み立て方

### 段階的導入の提案（必須）

一度に全部入れるのではなく、3段階で提案する:

1. **今すぐ（1週間以内）**: 無料ツール、既存機能の活用、設定だけで済むもの → Stage 1 から選定
2. **短期（1〜3ヶ月）**: 有料ツール導入、ワークフロー構築 → Stage 2 から選定
3. **中期（3〜6ヶ月）**: カスタム開発、システム連携、組織的な運用変更 → Stage 3 から選定

### 単体提案 vs 統合提案

業務A・B・Cに対してツールX・Y・Zを個別に提案するのではなく、まず「ツールXで A+B をまとめて改善できないか」を検討する。

## 最新ツール動向チェックリスト（提案時に必ず実施）

**このカタログの内容を鵜呑みにせず、提案を作成するたびに以下のWebSearchを実施して最新情報を反映する。** ここに記載のないツールが最適解になることも十分あり得る。

1. `「{業務カテゴリ} 自動化 ツール {現在の年} おすすめ」` で最新ツールを確認
2. `「{候補ツール名} 料金 {現在の年}」` で料金プランの変更を確認
3. `「{候補ツール名} 事例 活用」` で実際の導入事例を確認
4. 必要に応じて `「{候補ツール名} vs {比較ツール名}」` で比較情報を確認
5. AI関連ツールは特に変化が速いため、`「AI 業務自動化 最新 {現在の年} {現在の月}」` で直近のトレンドも確認する
