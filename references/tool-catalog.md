# AI・自動化ツールカタログ

Step 4（AI適用提案）で使用する。業務カテゴリ別に適用可能なツールを整理し、提案の質と一貫性を担保する。

## ツール選定の原則

1. **既存環境を優先する**: ユーザーが既にMicrosoft 365を使っているならPower Automate、Google Workspaceなら Apps Scriptを最初に検討する。新しいツールの導入は学習コストがかかるため、既存ツールの拡張で対応できないかをまず考える。
2. **無料→低コスト→有料の順に検討**: 同じことができるなら安い方を提案する。有料ツールを提案する場合は、無料代替の存在も併記する。
3. **組み合わせを意識する**: 1つのツールで複数の業務を改善できるなら、個別ツールを複数入れるより優先する。
4. **ITリテラシーに合わせる**: ヒアリングで得たユーザーのスキルレベルに応じて、ノーコード/ローコード/コードベースの提案を使い分ける。

## 時間削減の見積もり方法

削減時間は**控えめに見積もる**。過大な期待を持たせると信頼を失う。

### 見積もりの公式

```
月間削減時間 = 現状の月間作業時間 × 自動化率 × 定着率係数
```

- **自動化率**: references/analysis-framework.md の目安を使う
- **定着率係数**: 導入直後は 0.5〜0.7（慣れるまで時間がかかる）、定着後は 0.8〜0.9

### レンジで提示する

「月8時間削減」ではなく「月6〜10時間の削減見込み」のようにレンジで示す。根拠も1行で添える。

```
例: 「月次レポートのデータ集計部分を自動化することで、月6〜10時間の削減が見込まれます
    （現状: 月12時間 × 自動化率70% × 定着率0.7〜0.9）」
```

### やってはいけない見積もり

- 「AI導入で業務時間が半減」のような大風呂敷
- 対人業務の本質部分（交渉、判断）の時間を削減対象に含める
- 学習コスト・移行コストを無視した見積もり

## 導入手順テンプレート

各提案の「実装手順」は以下の構成で3〜5ステップにまとめる。

```
1. 準備: アカウント作成 / ツール導入 / 権限設定
2. 設定: ワークフロー構築 / テンプレート作成 / API連携
3. テスト: 小規模で試行（1週間程度）
4. 展開: チーム全体に展開 / マニュアル共有
5. 定着: 2週間後に振り返り / 微調整
```

個人パターンの場合はStep 4（展開）を省略。会社パターンの場合はStep 4に「部門間調整」を追加。

## 提案の組み立て方

### 単体提案 vs 統合提案

業務A・B・Cに対してツールX・Y・Zを個別に提案するのではなく、まず「ツールXで A+B をまとめて改善できないか」を検討する。

**統合提案の例**:
> Microsoft 365環境であれば、Power Automateで以下3業務を一括改善できます:
> - 日次売上データの自動転記（現状: 手作業30分/日）
> - 週次レポートの自動生成（現状: 手作業2時間/週）
> - 承認フローの電子化（現状: 紙回覧1時間/回）

### 段階的導入の提案

一度に全部入れるのではなく、3段階で提案する:

1. **今すぐ（1週間以内）**: 無料ツール、既存機能の活用、設定だけで済むもの
2. **短期（1〜3ヶ月）**: 有料ツール導入、ワークフロー構築
3. **中期（3〜6ヶ月）**: カスタム開発、システム連携、組織的な運用変更
