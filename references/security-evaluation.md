# セキュリティ評価ガイド

Step 5（セキュリティ評価）の詳細手順。提案した各ツールの情報セキュリティリスクを体系的に評価する。

## 評価の目的

ツールの「良い/悪い」を判定するのではなく、**ユーザーが判断するための材料を整理する**こと。最終的な導入判断はユーザーが行う。

---

## 調査手順

### 1. WebSearchで情報収集

各ツールについて以下の検索を行う:

```
検索クエリ例:
- "[ツール名] security policy"
- "[ツール名] SOC2 ISO27001"
- "[ツール名] data processing agreement"
- "[ツール名] AI training data opt-out"
- "[ツール名] セキュリティ 安全性"
```

**優先する情報源**:
1. ツールの公式セキュリティページ（/security、/trust）
2. 公式ドキュメントのプライバシーポリシー
3. SOC2レポートの公開サマリー
4. 信頼できる第三者レビュー（G2、Gartner等）

**避ける情報源**:
- 個人ブログの推測
- 古い情報（1年以上前）
- 競合ツールによる比較記事（バイアスあり）

### 2. 6項目の評価

各ツールについて以下を調査・記録する。

#### a. データの保存先

| 判定 | 基準 |
|------|------|
| 端末内のみ | データが外部に送信されない |
| 国内クラウド | 日本リージョンでのホスティング |
| 海外クラウド | US/EU等のリージョン |
| 不明 | 公開情報なし |

確認ポイント: リージョン選択の可否、データレジデンシー保証の有無

#### b. データの学習利用

| 判定 | 基準 |
|------|------|
| 利用なし | 明示的にオプトアウト or 学習に使わない宣言 |
| オプトアウト可 | デフォルトで学習利用あるが、設定で無効化可能 |
| 利用あり | ユーザーデータをモデル改善に使用 |
| 不明 | 明記なし |

これはAIツール特有の重要項目。特に以下を確認:
- API経由とWebアプリ経由で扱いが違うことがある（例: OpenAI APIはデフォルトでオプトアウト、ChatGPTはオプトイン）
- 「匿名化して利用」と「利用しない」は別物

#### c. 認証・アクセス制御

| 項目 | 確認内容 |
|------|---------|
| SSO | SAML/OIDC対応の有無、対応プラン |
| MFA | 二要素認証の対応方法（TOTP、SMS、ハードウェアキー） |
| RBAC | ロールベースのアクセス制御、カスタムロール |
| 監査ログ | 操作ログの取得・エクスポート可否 |

個人パターンではMFAの有無が主要ポイント。会社パターンではSSO・RBAC・監査ログまで確認。

#### d. コンプライアンス認証

| 認証 | 意味 | 重要度 |
|------|------|--------|
| SOC 2 Type II | セキュリティ管理の継続的監査 | 高（企業利用の基本） |
| ISO 27001 | 情報セキュリティマネジメント | 高 |
| GDPR | EU個人データ保護規則 | 中（EU関連の場合は高） |
| HIPAA | 米国医療データ保護 | 業種による |
| ISMAP | 日本政府のクラウド安全性評価 | 官公庁利用の場合は必須 |

#### e. データ暗号化

| 項目 | 確認内容 |
|------|---------|
| 転送時暗号化 | TLS 1.2以上 |
| 保存時暗号化 | AES-256等 |
| E2E暗号化 | エンドツーエンド暗号化の有無 |
| 鍵管理 | 顧客管理鍵（BYOK/CMK）の対応 |

#### f. リスクレベル総合判定

以下の基準で総合判定する:

| レベル | 基準 |
|--------|------|
| **低** | SOC2/ISO27001取得済、学習利用なし、国内リージョン選択可、MFA対応 |
| **中** | 一部認証あり、学習利用はオプトアウト可、暗号化対応 |
| **高** | 認証なし、学習利用あり/不明、データ保存先不明 |
| **確認不可** | 十分な公開情報がない |

## 規模パターン別の評価重点

### 個人パターン
重点: データの学習利用、基本的な暗号化
軽減可能: SSO、監査ログ、コンプライアンス認証（ただし個人事業主で機密データを扱う場合は例外）

### チームパターン
重点: 認証・アクセス制御（MFA + 最低限のRBAC）、データの学習利用、暗号化
軽減可能: 高度なコンプライアンス認証（SOC2推奨だが必須ではない）

### 会社パターン
重点: 全項目を網羅的に評価。特にSSO、監査ログ、SOC2/ISO27001は必須レベル
追加確認: データ処理契約（DPA）の締結可否、SLA、インシデント通知体制

---

## 調査で情報が見つからない場合

「確認不可」と明記し、以下のアドバイスを添える:

- 導入前にベンダーに直接問い合わせることを推奨
- 確認すべき具体的な質問リストを提示
- 代替ツール（セキュリティ情報が公開されているもの）があれば併記

```
例: 「〇〇のデータ学習利用ポリシーは公開情報から確認できませんでした。
    導入検討時にベンダーへ以下を確認することを推奨します:
    - ユーザーデータのAIモデル学習への利用有無
    - オプトアウトの可否と方法
    - データ保持期間と削除ポリシー」
```

## レポートへの反映

セキュリティ評価結果はHTMLレポート・PPTXの両方で以下の形式で表示する:

### 一覧テーブル（全ツール横並び比較）

| 項目 | ツールA | ツールB | ツールC |
|------|--------|--------|--------|
| データ保存先 | 国内クラウド | 海外クラウド | 端末内 |
| 学習利用 | なし | オプトアウト可 | なし |
| 認証 | SSO/MFA | MFA | なし |
| コンプライアンス | SOC2, ISO27001 | SOC2 | — |
| 暗号化 | TLS 1.2 + AES-256 | TLS 1.2 | — |
| **リスクレベル** | **低** | **中** | **低** |

### 補足コメント

各ツールに1〜2行の補足を付ける。特にリスクが「中」「高」の場合は、具体的に何がリスク要因かを明記する。
