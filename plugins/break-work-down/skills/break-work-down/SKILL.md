# 業務分析・改善提案スキル

現場の業務内容を分析し、効率化・自動化の改善を提案するスキル。各フェーズはサブエージェントに委譲し、オーケストレーターは設定管理・ユーザーとの対話・フェーズ間のデータ受け渡しに集中する。

## 前提

- アウトプットの質はインプットの質で決まる。可能な範囲でデータを収集し、足りない部分は最小ヒアリングで補完する
- 業務をタスクレベルまで把握し、具体的な工程に対して具体的な手段を提案する
- 業務の価値を否定しない。「この作業の目的を、別の手段で達成できないか」という姿勢を貫く
- **選択肢の提示には必ず AskUserQuestion ツールを使う**（テキスト箇条書き禁止）

## `<skill-base-dir>` の解決

スキル開始時に Glob ツール等で SKILL.md の位置を特定し、そのディレクトリパスを `<skill-base-dir>` として以降すべてのパス参照に使用する。

---

## ワークフロー概要

```
Step 0: 初期化（deferred tools取得 + 設定読込）
Step 1: 許可取得 + 情報収集     → agents/collect.md に委譲
Step 2: 分析 + ヒアリング       → agents/analyze.md に委譲
Step 3: 調査 + 優先順位付け     → agents/research.md に委譲
Step 4: レポート出力            → agents/report.md に委譲
```

---

## Step 0: 初期化

### 0a. Deferred tools の事前取得

サブエージェントとの通信やユーザーへの質問に使う deferred tools のスキーマを、**最初に ToolSearch で取得しておく**:

```
ToolSearch("select:AskUserQuestion,SendMessage", max_results=2)
```

これにより、以降のステップで `AskUserQuestion`（ユーザーへの質問）と `SendMessage`（サブエージェントへの追加情報送信）を正しいパラメータで呼び出せる。

### 0b. 設定の読み込み

Read ツールで `data/config.json` を読み込む。

- **存在しない場合（初回）**: `PYTHONUTF8=1 python "<skill-base-dir>/scripts/save_collection_data.py" reset` で初期化
- **存在する場合**: 前回の設定を再利用。未設定の項目のみ Step 1 で取得

---

## Step 1: 許可取得 + 情報収集

### 1a. ツール検出

利用可能なMCPサーバーとローカルのブラウザ履歴を確認する。

### 1b. データ収集の許可取得

**自動収集データがあるほどヒアリング負荷が下がり、提案の精度も上がる。** 収集先の許可は必ず **2方向（接続済みツール + ブラウザ履歴）** で聞く。以下の順に進める。

#### ① 未接続ツールの接続案内

主要ツール（Slack, Notion, Google Workspace等）のうち **未接続のもの** がないか確認し、接続を提案する。一部が接続済みでも、未接続の主要ツールがあれば案内する。

**AskUserQuestion で聞く内容:**
- 普段使っている業務ツールで、まだ接続されていないものはあるか（Slack / Notion / Google Workspace / Telegram / Discord 等）
- 接続を試すか、今ある接続だけで進めるか

**ユーザーがMCP接続を希望した場合**: 該当ツールの接続手順を案内する。
- Slack: `claude plugin install slack` → OAuth認証
- Notion: `claude plugin install notion` または `claude mcp add notion -- npx -y @notionhq/notion-mcp-server`
- Google Workspace: `claude mcp add google-workspace -- npx -y @googleworkspace/cli mcp`
- 接続後、ツールがセッションに反映されるまで再起動が必要な場合がある。接続完了後に Step 1a に戻って検出をやり直す

**接続を追加しない場合**: そのまま②に進む。

#### ② 収集先の許可 + 制約条件の取得（AskUserQuestion 1回）

以下を1回の AskUserQuestion（最大4問）にまとめて聞く。`config.json` で設定済みの項目はスキップし、4問に収める。

**必須: 収集先の許可（2問）**

- **接続済みツールからの収集許可**: Step 1a で検出した**全ての**接続済みMCPツールを選択肢に並べ、どれを収集対象にするか聞く。「収集しない」も選択肢に含める。ツールの種類は問わない（コミュニケーション: Slack, Discord / ナレッジ管理: Notion / グループウェア: Google Workspace / プロジェクト管理: GitHub, Wrike / ワークフロー自動化: n8n, Zapier 等）。代表的なツール以外でも、検出されたMCPツールは全て選択肢に含めること。特にワークフロー自動化ツールが接続されている場合は、既存ワークフローの一覧取得により既に自動化済みの業務を把握できるため提案の精度が大きく向上する旨を説明する。
- **ブラウザ履歴からの収集許可**: 使用ブラウザ（Chrome / Edge / 両方）を選択させる。「収集しない」も選択肢に含める。

**残り枠で: 制約条件（未設定項目のみ、最大2問）**

- **分析対象の単位**: 個人 / チーム・部門 / 特定プロジェクト
- **予算の目安**: 追加費用ゼロ / 月額数千円 / 月額万円以上 / 未定
- **組織種別**: NPO・非営利 / 公的機関 / 大企業 / スタートアップ / フリーランス / 該当なし

4問に収まらない制約条件は次の AskUserQuestion で聞く。

```
例: AskUserQuestion（4問にまとめた場合）
Q1: 「接続済みのSlack・GitHub等からデータを自動収集してよいですか？」
  - Slack + GitHub 両方OK（推奨）
  - Slack のみ
  - GitHub のみ
  - 収集しない

Q2: 「ブラウザの閲覧履歴から業務ツールの利用パターンを分析してよいですか？」
  - Chrome の履歴を使う
  - Edge の履歴を使う
  - 両方の履歴を使う
  - 収集しない

Q3: 「分析対象の範囲はどれですか？」
  - 個人 / チーム・部門 / 特定プロジェクト

Q4: 「改善施策にかけられる予算の目安は？」
  - 追加費用ゼロ / 月額数千円 / 月額万円以上 / 未定
```

**両方の収集先で「収集しない」の場合**: ヒアリング中心モードに切り替える。

取得した設定を保存:
```bash
echo '{"analysis_scope": "individual", "budget_constraint": "zero", "organization_type": "freelance", "browser": "chrome"}' | PYTHONUTF8=1 python "<skill-base-dir>/scripts/save_collection_data.py" save-config
```

### 1c. 収集エージェントの起動

**収集対象がゼロの場合（MCP未接続 + ブラウザ履歴も拒否）**: 収集エージェントをスキップし、Step 2 をヒアリング中心モードで起動する。

**収集対象がある場合**: Agent ツールで収集エージェントを起動する。

```
Agent ツール起動:
  プロンプトに含める情報:
  - agents/collect.md の絶対パスと「Read ツールで読んでから収集を始めること」
  - <skill-base-dir> の値
  - config.json の内容
  - permitted_sources のリスト
  - 「ユーザーは既にアクセスを許可済みである」
  - 「ユーザーへの追加の許可確認・質問は一切行わないこと」
```

収集エージェントの結果（JSON）を受け取り、Step 2 に渡す。

---

## Step 2: 分析 + ヒアリング

Agent ツールで分析エージェントを起動する。収集データが空の場合は `hearing_only: true` をプロンプトに含め、ヒアリング中心モードで起動する。

```
Agent ツール起動:
  プロンプトに含める情報:
  - agents/analyze.md の絶対パスと「Read ツールで読んでから分析を始めること」
  - <skill-base-dir> の値
  - 収集エージェントの結果JSON（空の場合は `{}` + `hearing_only: true`）
  - config.json の内容
```

分析エージェントは必要に応じて AskUserQuestion でヒアリングを行い、業務一覧・分類・ボトルネックを特定して返す。

### フォールバック: 分析エージェントがヒアリングを実行できなかった場合

分析エージェントの結果に `pending_questions` フィールドが含まれている場合、分析エージェントが AskUserQuestion を使えなかったことを意味する。この場合、オーケストレーター側で以下を行う:

1. `pending_questions` の内容をもとに AskUserQuestion でユーザーにヒアリングを実施
2. ヒアリング結果と元の入力データを合わせて、分析エージェントを再起動する（`pending_questions` を返した時点でエージェントは終了済みのため、SendMessage ではなく再起動）

---

## Step 3: 調査 + 優先順位付け

Agent ツールで調査エージェントを起動する。

```
Agent ツール起動:
  プロンプトに含める情報:
  - agents/research.md の絶対パスと「Read ツールで読んでから調査を始めること」
  - <skill-base-dir> の値
  - 分析エージェントの結果JSON
  - config.json の内容
  - 「ユーザーへの質問は一切行わないこと」
```

調査エージェントは3方向の並列調査を行い、優先順位付きの提案を返す。

---

## Step 4: レポート出力

Agent ツールでレポートエージェントを起動する。

```
Agent ツール起動:
  プロンプトに含める情報:
  - agents/report.md の絶対パスと「Read ツールで読んでから生成を始めること」
  - <skill-base-dir> の値
  - 分析エージェントの結果JSON
  - 調査エージェントの結果JSON
  - config.json の内容
  - 「ユーザーへの質問は一切行わないこと」
```

レポートエージェントはHTMLレポートを生成し、ファイルパスを返す。

### レポートの提示（必ずワークフローの最後に行う）

**この提示がスキル全体の最終アウトプットになる。** 他の補足説明や次ステップの案内より後に、以下を必ず実行する。

1. **ブラウザで自動オープン**: Bash ツールで `start "" "<ファイルパス>"` を実行し、デフォルトブラウザでレポートを開く（Windows）。Mac の場合は `open`、Linux の場合は `xdg-open` を使う。
2. **ファイルパスを明示**: 自動オープンの成否にかかわらず、レポートの絶対パスをユーザーに提示する。

**HTTPサーバーは起動しない**（不具合で開けないケースが多いため）。

## ガードレール

- **現場の実情を十分に把握してから提案に進む** — 不十分な場合は「暫定提案」として不確実性を明記
- **提案には根拠を明記する**: 収集データからの根拠に加え、調査エージェントがWeb検索で見つけた参照URL（ツール紹介記事・公式ドキュメント等）もレポートに掲載する
- **外部ツールへのアクセスは必ずユーザーの許可を得てから**
- **KPI数値は内訳合計と一致させる**
- **プライバシー/安全性**: ユーザーの生データ（個人情報・生メッセージ・閲覧URL・会社名・個人名・サービス固有名）をレポートに含めない。法定守秘義務がある職種では自動収集を原則スキップ。※ 調査エージェントが検索した公開Webページの参照URLはこの制約の対象外
- **未許可領域の扱い**: 許可がないデータ領域は根拠として使わない
- **非デジタル業務が主体の場合**: 自動収集をスキップしヒアリング起点にする
