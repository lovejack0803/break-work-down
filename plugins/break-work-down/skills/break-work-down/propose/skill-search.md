# スキル・エージェント探索

ボトルネック業務に対して、導入可能な既製のエージェントスキルを探す。Web調査とは独立した調査方向であり、必ず並行で実行する。

## 探索ソース

| # | ソース | 規模 | 検索手段 | 特徴 |
|---|---|---|---|---|
| 1 | [skills.sh](https://skills.sh) | — | CLI `npx skills find` / Web | Vercel製。インストール実績ランキングあり |
| 2 | [skillsmp.com](https://skillsmp.com) | 17万5千件+ | CLI `skills search` / Web | 最大規模のマーケットプレイス |

両方を**必ず**検索する。1ソースだけで終わらない。

## 探索手順

### 1. skills.sh（Vercel製CLI）

```bash
npx skills find <キーワード>    # FZF風インタラクティブ検索
npx skills add <owner/repo>     # インストール
npx skills list                  # インストール済み確認
npx skills check                 # アップデート確認
```

- WebFetchで `https://skills.sh` のランキングも確認し、実績のあるスキルを把握する
- ボトルネック業務に関連するキーワードを**最低3つ**検索（英語で: editing, proofreading, workflow 等）
- 検索結果が0件のキーワードも記録する（「該当なし」の根拠として使う）

### 2. skillsmp.com（Agent Skills CLI）

```bash
# 未インストールなら初回のみ
npm install -g agent-skills-cli

skills search <キーワード>       # キーワード検索（複数選択でインストール可）
skills search -i                 # FZF風インタラクティブ検索
skills install @author/name      # インストール
skills check                     # インストール済み確認
```

- CLIが使えない場合は Web UI（`https://skillsmp.com`）で検索
- 403エラー時は `site:skillsmp.com [キーワード]` でWeb検索

## 評価基準

| 観点 | 確認内容 |
|---|---|
| **業務適合性** | ボトルネック業務の目的に合致しているか |
| **言語対応** | 日本語に対応しているか（中国語・英語専用でないか） |
| **導入コスト** | インストールだけで使えるか、カスタマイズが必要か |
| **実績** | インストール数が十分か（目安: 100以上で検討対象） |
| **保守状態** | 最終更新が古すぎないか |

## 出力フォーマット

```
## スキル探索結果

### 直接導入可能なスキル
（見つかった場合: スキル名・インストールコマンド・概要・適合度評価）
（見つからなかった場合: 「該当なし」+ 検索したキーワード一覧）

### 参考にできるスキル
（直接導入はできないが、フレームワーク構造やプロンプト設計が参考になるスキル）

### 検索ログ
（各ソースでの検索キーワードと結果件数）
```

## よくある判断ミス

| # | 問題 | 対策 |
|---|---|---|
| 1 | **英語スキルの日本語適用** — 多くのスキルは英語前提。「copy-editing」スキルをそのまま使うと英語の文体ルールが適用される | 日本語対応の確認は必須。READMEやSKILL.mdに言語の記載がなければ英語専用と判断 |
| 2 | **スキルの過信** — 「できる」と謳っていても業務に完全フィットするとは限らない | 導入前に小さなテストケースで検証することを提案に含める |
| 3 | **skillsmp.comのアクセス制限** — 403エラーを返す場合がある | Web検索で `site:skillsmp.com` や `skillsmp [キーワード]` で間接的に収集 |
| 4 | **CLIのコマンド競合** — skills.sh と agent-skills-cli は両方 `skills` コマンドを使う | 同一環境に両方入れる場合は `npx skills`（skills.sh）と `skills`（skillsmp）で使い分ける |
| 5 | **実行環境の前提** — CLIツール（npx, npm）はNode.js環境が必要 | ユーザーの端末にNode.jsがない場合は、WebUI（`skills.sh`, `skillsmp.com`）での検索に切り替える。CLI実行を試みる前に環境確認を行う |
| 6 | **技術力の高いユーザーへの価値** — 技術力が高いユーザーはGitHub/npm/PyPIで直接探す方が早いと感じる | 探索結果の価値は「知らなかったスキルを発見する」ことにある。ユーザーが既に知っていそうな有名スキルを除外し、ニッチだが有用なスキルを優先的に提示する |
