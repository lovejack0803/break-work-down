# 業務分析フレームワーク 詳細

## 分類の判定フロー

```
業務を1つ取り出す
  ├─ 手順が定型化されている？
  │   ├─ Yes → 定常業務
  │   └─ No → 発生が予測可能？
  │       ├─ Yes（低頻度だが手順あり）→ 定常業務
  │       └─ No → 例外業務
  └─ 本質的に人間の判断・感情対応が必要？
      ├─ Yes → 対人業務
      └─ No → 上記フローで判定
```

対人業務の周辺作業（議事録、日程調整、資料準備など）は別の業務として切り出し、定常業務として再分類する。

## 費用対効果スコアの算出

```
削減見込み = 年間工数 × 自動化可能割合

ランク判定:
  削減見込みの規模感に応じてA/B/Cにランク分けする。閾値は対象の業務規模に合わせて調整する
```

## 業務削減の判定（Step 4で使用）

業務削減の判定基準はSKILL.md Step 4を参照。

### 削減効果の過大評価に注意

「この作業をゼロにできる」という廃止提案で、実際には「完全にゼロにはならないが大幅に減る」ケースが多い。例: 紙の申請書廃止→SmartHR自己申請に切り替えても、従業員からの問い合わせ対応は残る。削減効果は「残存作業」を差し引いて算出すること。

## 優先順位の決め方

ランクだけでなく、以下も加味して最終的な優先度を決める:

1. **業務削減（廃止・統合・頻度削減）**: コストゼロで効果が出るもの → 最優先
2. **クイックウィン**: 導入難易度「低」× ランクA → 次に優先
3. **ROI重視**: 削減時間が大きい順
4. **波及効果**: 1つのツール導入で複数業務が改善されるもの
5. **ユーザーの困り度**: ヒアリングで「しんどい」と言われた業務を優先

導入難易度は「技術的難易度」と「組織的難易度（承認不要/部門長承認/全庁的調達）」の2軸で評価する。公的機関・大企業では技術的に容易でも組織的に半年以上かかることがある。

## 分析対象の特性による調整

分析の基本フローはSKILL.md Step 4の通りだが、対象の特性によって評価軸や粒度を調整する必要がある。以下のルールを判定フロー・費用対効果算出・優先順位付けの各段階で適用する。

| 特性 | 調整ルール |
|---|---|
| **非デスクワーク業種**（飲食・小売・製造・物流等） | フィジカルな業務は「対人」に見えがちだが、周辺作業（発注、シフト管理、日報等）を切り出して改善対象にする。「自動化可能割合」ではなく「情報伝達・記録・報告の効率化割合」で評価する |
| **管理職が対象** | 本人の業務（判断・承認）は自動化しにくい。「本人の業務」ではなく「関与するプロセス全体」に視点を広げ、承認待ち短縮やレポート自動生成等の周辺改善を探る |
| **時期依存のボトルネック**（年度末・期末等） | 月次平均ではなくピーク期間の工数で評価。業務一覧に「ピーク時期」列を設け年間負荷分布を可視化する |
| **1人運営**（フリーランス・個人事業主） | 業務数が多くなりすぎる。時間消費の大きい上位3業務に絞って深掘りし、残りは一覧記録に留める |
| **既に自動化済みの業務** | 「完了」として除外せず、(1) エラー率 (2) メンテナンス工数 (3) 未カバーの例外を確認する。改善は新規導入ではなく既存の拡張・統合・AI強化にある可能性が高い |
| **デジタル化が未着手** | 「最終形」だけでなく「最初の一歩」を含めた段階的ロードマップを設計する。例: 手書き帳簿→スマホ撮影→freee取込→自動仕訳 |

## よくある判断ミス

### 1. 分類の曖昧さ

「例外業務」と「定常業務」の境界があいまいになりがち。以下のルールで判定する:
- **定常業務**: 手順書が書ける程度に定型化されている、または低頻度でも発生が予測可能（月次決算、入退社手続等）
- **例外業務**: 手順が毎回異なる、または発生を予測できない（クレーム対応、システム障害等）
- **対人業務**: 本質的に人間の判断・感情対応が必要（採用面接、顧客折衝等）。ただし「対人業務の周辺作業」（議事録作成、日程調整）は定常業務として切り出す

「勤怠不備の差し戻し」は例外に見えるが、「毎月必ず発生する＋手順が定型化できる」なら定常。発生頻度と手順の定型度で判断する。

カスタマーサクセスや問い合わせ対応部門では、1件1件の対応は例外業務に見えるが、対応フロー全体（受付→分類→対応→エスカレ→クローズ）は定常業務。分類の粒度を間違えると全てが例外に分類され、改善提案が出せなくなる。

### 2. 自動化可能割合の見積もり

自動化可能割合は業務の具体的な内容次第で大きく変動する。固定値ではなく、対象業務の実態に基づいて推定すること。
