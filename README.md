# break-work-down

業務データを自動収集し、工程レベルの改善提案レポートを生成するClaude Codeスキルです。

ヒアリングから始めるのではなく、実データから実態を把握するアプローチを採用した「現場目線の業務改善スキル」。Slack・Google Workspace・ブラウザ履歴などから、エージェントが先回りして業務内容を自動収集します。ユーザーの説明負担を減らしながら、**本人も気づいていなかった業務の実態**を浮かび上がらせます。

本スキルはエージェントの能力を「良質なインプットを作ること」に全振りしているのが大きな特徴です。インプットの質がアウトプットの質を決める——これが本スキルの根底にある考え方です。

つまり、このスキルは他のスキルを活用するのに役立つ頼もしい土台であり、**スキル活用の出発点**です。単体での活用がもちろん、他のスキルと組み合わせたときに最大の価値を発揮します。

## インストール

Claude Code で以下を実行してください:

```
/plugin marketplace add lovejack0803/break-work-down
/plugin install break-work-down@break-work-down
```

## 使い方

以下のように話しかけるだけで起動します:

```
業務を分析して改善提案をしてほしい
うちの業務を見てほしい
何か効率化できないか
```

スラッシュコマンドでも起動できます:

```
/break-work-down:break-work-down
```

### 実行の流れ

1. **データ収集の許可取得** — 接続済みの MCP ツールやブラウザ履歴から、どの情報を使ってよいか確認します
2. **自動収集** — 許可されたソースから業務データを並列収集します（MCP 未接続の場合はヒアリングに切り替え）
3. **分析 + ヒアリング** — 業務一覧を構築し、不足情報だけを質問します
4. **改善手段のリサーチ** — ツール・AI 活用・既製スキルの 3 軸で並列調査します
5. **HTML レポート出力** — 優先順位付きの改善提案レポートをブラウザで開きます

### 対応ツール

Slack / Google Workspace / Notion / GitHub / Discord / Telegram など、MCP 接続済みのツールに対応します。Chrome・Edge のブラウザ履歴も収集可能です。MCP 非対応の Web アプリについては、Browser Use CLI 2.0 経由で画面情報を取得できます。
